**Exam Technicalities** 

**Format:** 20 MCQ + 4 FRQ

**Coding by Hand**

- Either C or C++ is acceptable #927
- Helper functions are recommended
- If you need to insert lines into already-written code, make it clear where they go — e.g., draw the new code in a box with an arrow pointing to the insertion point #937

**Syntax & Style**

- Write code as close to the lab implementations as possible
- `using namespace std;` is allowed #927
- `systemd`'s PID is 1 #954
- You don't need to write `#include` statements #959
- Write commands with all parameters filled in #1008
- Restore `stdin`/`stdout`/`stderr` if you redirect them #965
- The only helper functions you can assume are implemented are the ones provided on the exam #972
- Error handling is expected — if you encounter an error, exiting is a safe action to take #934
- Declaring helper function prototypes before `main` is best practice #932

**Miscellaneous**

- If you need more paper, ask a TA #973
