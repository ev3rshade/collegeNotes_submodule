# Interrupt
(asynchronous processing)
an event that requires immediate attention

when received, CPU will stop current task and jump to 'interrupt handler' that handles that specific interrupt

allows computer to switch between [[User Mode]] and [[Kernel Mode]]
**RUNS IN KERNEL MODE** 
- an interrupt handler must read device/CPU registers and execute instructions only available in kernel mode
- interrupt vector can be modified only in kernel mode
- interrupt vector initialized during boot time; modified when drivers added to system


---
## Steps of Servicing an Interrupt

1. The CPU saves the PC and registers in an execution stack
2. CPU looks up the corresponding interrupt handler in the interrupt vector
3. CPU jumps to interrupt handler and runs it
4. CPU restores the registers and returns back to the place in the program that was interrupted
5. The program continues execution as if nothing happened

\* in some cases CPU retries the instruction that was interrupted


**Example:**
1. OS request Device Operation (write to disk)
2. Device Runs Operation | OS does other things in parallel with device
3. When Operation is complete interrupt OS
4. OS services interrupt and continues

---
## Types of Interrupts

**1 Device Interrupts**
generated by devices when a request is complete or an event that requires CPU attention happens

mouse is moved | key is typed | Ethernet packet arrives | the hard drive has completed a read/write | CD has been inserted int the CD drive
